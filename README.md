# Flow Worker

`Flow Worker`는 새 독립 프로그램입니다.

이 프로젝트의 기준은 아주 단순합니다.

- `Flow Classic Plus`의 `이미지 워커`
- `Flow Classic Plus`의 `S자동화 워커`

오직 이 두 개만 참고해서, 두 워커를 가볍게 합친 새 앱을 만드는 것이 목적입니다.

현재 사용자 기준 용어는 이렇게 잡습니다.

- `이미지 워커` -> `이미지 모드`
- 예전 `S자동화 워커` -> `비디오 모드`
- 다만 `비디오 모드`의 실제 작동 방식은 기존 `S자동화 워커`를 그대로 따라갑니다.

## 현재 상태

- 독립 저장소와 독립 실행 구조는 남겨둔 상태입니다.
- 기준은 이제 `이미지 워커 / S자동화 워커` 두 개로 고정했습니다.
- 현재는 `FlowWorker` UI가 뒤에서 숨은 원본 워커 엔진을 IPC로 제어하는 브리지 방식으로 동작합니다.
- 이미지 모드는 실제 사용자 테스트에서 `생성 -> 레퍼런스 첨부 -> 다운로드 저장`까지 한 번 성공 확인했습니다.
- 대기열은 이제 그록워커처럼 여러 카드가 한 줄에 보이는 구조로 다시 정리했고, 초기 색도 파란 톤이 아니라 브라운/앰버 계열로 분리했습니다.
- 비디오 프롬프트 경로는 새 규칙으로 연결해 둔 상태이고, 다음 실사용 검증 대상입니다.
- `FlowWorker_실행.vbs`를 다시 누르면 이제 자동으로 `Flow Worker2`, `Flow Worker3`처럼 새 워커가 따로 뜨도록 멀티 실행 기반을 넣었습니다.
- 각 워커는 `browser_profile_name`과 `browser_attach_url`을 따로 가져, 병렬 작업 때 프로필/포트가 겹치지 않게 분리합니다.

## 실행

Windows:

- `FlowWorker_실행.vbs`
- 또는 `python -m flow_worker.main`

## 현재 폴더 구성

- `flow_worker/config.py`
- `flow_worker/prompt_parser.py`
- `flow_worker/browser.py`
- `flow_worker/automation.py`
- `flow_worker/ui.py`

## 개발 원칙

- `Flow Classic Plus 전체`를 기준으로 삼지 않습니다.
- 메인 통합 앱, 이어달리기, 기타 안 쓰는 기능은 참고 대상이 아닙니다.
- 새 기능을 넣을 때는 먼저 `이미지 워커`와 `S자동화 워커` 안에서 실제 동작을 확인한 뒤 옮깁니다.

## 현재 반영한 프롬프트 규칙

- 이미지 프롬프트:
  - `S001 Prompt : ...`
  - `S001>S002 Prompt : ...`
  - 실제 Flow 입력은 둘 다 `S001 Prompt : ...` 형태로 정리합니다.
- 비디오 프롬프트:
  - `V001 Prompt : ...`
  - `V001>V002 Prompt : ...`
  - 실제 Flow 입력은 `V001 Prompt : ...` 형태로 정리합니다.
  - 대신 라우트 정보는 따로 보존해서, 내부 브리지에서는 `S001>S002 Prompt : V001 Prompt : ...` 같은 임시 런타임 파일로 바꿔 원본 비디오 워커 흐름에 넘깁니다.
  - 즉 사용자 기준 프롬프트는 `V` 태그를 유지하고, 원본 워커가 요구하는 시작/끝 프레임 정보만 내부에서 `S` 태그로 따로 실어 보냅니다.
  - 비디오 대기열 표시와 다운로드 검색은 `V001`, `V001>V002` 기준으로 맞추는 쪽으로 보강 중입니다.
