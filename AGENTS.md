# FlowWorker 작업 규칙

## 기본 원칙
- 사용자는 한국어로 소통한다.
- 코드/파일 수정 전에는 무엇을 바꿀지 짧게 먼저 말한다.
- “완료”라고 말할 때는 실제로 확인한 범위만 말한다.
- 사용자는 감으로 만든 기능보다 실제 Flow 화면, 실제 파일, 실제 로그 기준 검증을 원한다.
- `git reset --hard`, `git checkout --` 등 사용자 변경을 날리는 명령은 금지한다.
- Codex/루프PD가 콘텐츠 생산 목적으로 Flow 사이트를 직접 반복 조작하지 않는다. 반복 생성 실행은 반드시 FlowWorker를 통해서만 한다.

## FlowWorker 기준
- 현재 FlowWorker는 `Flow Classic Plus`를 숨겨 실행하거나 브릿지로 연결하지 않는다.
- 기준 엔진은 독립 `flow_worker.automation.FlowAutomationEngine`이다.
- 원본 `Flow Classic Plus`는 참고/학습용이며, 연결 대상이 아니다.
- Flow UI는 자주 바뀌므로 selector만 믿지 말고 실제 DOM 후보, geometry, action trace를 같이 확인한다.
- 실패 분석은 최신 `logs/action_trace_*.log`를 먼저 본다.
- Flow 사이트 직접 수동 조작은 기본적으로 사용자 담당이다. Codex는 자동화 안전성을 위해 FlowWorker 코드/설정/로그/파일을 우선 다룬다.
- 예외: FlowWorker 기능 수정/강화/업그레이드 검증을 위한 제한적 Flow 화면 관찰과 소량 테스트는 가능하다.
- 금지: Codex가 콘텐츠 생산 목적으로 프롬프트 입력/생성을 하루 수십 번 반복하는 방식.
- 실제 생성은 FlowWorker 앱의 워커 프로필/포트/저장폴더/프롬프트 파일을 통해 실행한다.
- 비디오 롱폼 루프의 핵심 기능은 `생성 후 확장`, `현재 영상 확장`, `마지막 프레임 자동저장`, `다음 시작 프레임 추출`이다.

## 롱폼 사이드 프로젝트
- 사이드 프로젝트명: `몽글루프 스튜디오`
- 대표 캐릭터 1차 이름: `몽글이`
- Codex 총괄 프로듀서 별칭: `루프PD`
- 목표: Flow/Veo와 Suno 음악을 조합해 3-4분짜리 귀엽고 반복 시청 가능한 롱폼 영상을 제작한다.
- 초반 10편 이상은 먹방 ASMR 중심으로 제작한다.
- 기본 음악 방향은 잔잔한 로파이 + 약한 ASMR 분위기다.
- Codex 역할: 단순 자동화 개발자가 아니라 `총괄 프로듀서/감독`이다.
- 사용자가 프롬프트를 잘 못 짜도 괜찮게, Codex가 먼저 콘셉트, 카메라, 장소 동선, 감정 언어, 반복 캐릭터 행동, 장면 목적을 설계한 뒤 실행용 프롬프트로 변환한다.
- 사용자가 아이디어와 취향을 주면, Codex는 그것을 제작 가능한 영상 기획/샷 리스트/Flow 프롬프트/검증 루프로 정리한다.
- 우선순위는 롱폼이다. 숏츠는 고려하되, 먼저 16초 체인을 이어 3-4분 영상으로 완성한다.
- 핵심 제작 루프:
  1. 시작 이미지 또는 시작 프레임을 준비한다.
  2. Flow에서 8초 영상을 만든다.
  3. `확장` 전에 모델을 `Veo 3.1 - Fast`로 맞춘다.
  4. 확장해서 16초 영상을 만든다.
  5. 16초 영상을 다운로드한다.
  6. 다운로드 영상의 마지막 프레임을 `S다음번호.png`로 추출한다.
  7. 그 이미지를 다음 시작 프레임으로 사용해 다음 16초를 만든다.
  8. 이 과정을 반복해 3-4분 롱폼으로 이어붙인다.
- 구현은 “생성 버튼 누르기”보다 “실제로 이어지는 영상이 나오는 제작 파이프라인”을 기준으로 설계한다.

## 제작 판단 기준
- 먹방형은 “한입씩 줄어드는 음식, 미세한 표정, 손/입 움직임, 안정된 카메라”가 중요하다.
- 여행형은 프롬프트보다 먼저 “동선, 서사, 감정선, 장소별 장면 목적”을 잡는다.
- Suno 음악은 배경 감정선과 반복 시청성을 만든다.
- 캐릭터는 한 프로젝트 안에서 일관성을 유지해야 한다.
- 장면 전환은 마지막 프레임 체인으로 자연스럽게 이어지는 것을 우선한다.
