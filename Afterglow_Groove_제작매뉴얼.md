# Afterglow Groove 제작 매뉴얼

## 핵심
원곡을 만들고, 그 곡에 맞는 단편 뮤직비디오 컷을 FlowWorker로 제작한 뒤, 사람이 편집해 완성하는 음악 채널이다.

## 역할
- 사용자는 음악 방향과 취향을 말한다.
- 그루브PD는 총괄 프로듀서로서 곡 콘셉트, Suno 프롬프트, 가사, 영상 플롯, FlowWorker 프롬프트를 만든다.
- FlowWorker는 그 프롬프트를 실행하는 직원이다.

## 사용 도구
- Suno: 원곡 제작
- FlowWorker: 이미지/비디오 컷 제작
- CapCut 또는 편집툴: 음악과 영상 조립
- FFMPEG: 라이브용 루프 파일 송출 준비

## 제작 순서
1. 사용자가 음악 방향을 말한다.
2. 그루브PD가 곡 콘셉트를 정리한다.
3. 그루브PD가 Suno 프롬프트와 가사를 작성한다.
4. 사용자가 Suno에서 후보를 생성한다.
5. 그루브PD와 사용자가 가장 좋은 곡을 고른다.
6. 그루브PD가 곡의 BPM, 분위기, 가사 키워드를 기록한다.
7. 그루브PD가 뮤직비디오 시놉시스를 작성한다.
8. 그루브PD가 8초 단위 컷 리스트를 작성한다.
9. 그루브PD가 FlowWorker용 S컷 이미지 프롬프트를 작성한다.
10. 그루브PD가 FlowWorker용 V컷 영상 프롬프트를 작성한다.
11. FlowWorker로 컷을 생성한다.
12. 좋은 컷만 선별한다.
13. 편집툴에서 음악에 맞춰 조립한다.
14. 롱폼 업로드본을 만든다.
15. 라이브용 10-30초 루프 영상을 만든다.
16. Shorts/Reels 파생본을 만든다.

## 서사 설계
Afterglow Groove 영상은 가사 내용 그대로 찍는 뮤직비디오가 아니다.
곡의 리듬과 정서를 바탕으로 독립적인 단편 영화 플롯을 만든다.

예:
- 가사가 사랑을 말해도, 영상은 퇴근 후 같은 아파트 복도에서 매일 엇갈리는 두 사람의 이야기일 수 있다.
- 가사가 밤의 도시를 말해도, 영상은 낡은 자동차 뒷좌석에 남겨진 카세트테이프를 중심으로 흘러갈 수 있다.
- 가사가 그리움을 말해도, 영상은 누군가가 오래된 스토리보드 노트를 넘기며 기억을 재조립하는 구조일 수 있다.

## 이어가기와 끊기
FlowWorker의 마지막 프레임 추출 기능은 계속 사용한다.
다만 모든 컷을 반드시 이어붙이지 않는다.

### 이어가기 컷
```text
V001 다운로드
-> 마지막 프레임 추출
-> S002로 저장
-> S002를 시작 프레임으로 V002 생성
```

사용할 때:
- 같은 장면이 감정적으로 계속 이어져야 할 때
- 인물의 미세한 행동이 중요할 때
- 빈티지 자동차, 복도, 바 테이블처럼 공간의 지속성이 매력일 때

### 새 장면 컷
```text
V003 종료
-> 새 이미지 S004 생성
-> S004를 시작 프레임으로 V004 생성
```

사용할 때:
- 다른 장소로 넘어갈 때
- 회상/상상/스토리보드 컷이 필요할 때
- 시각적 환기가 필요할 때
- 반복 템플릿처럼 보일 위험이 있을 때

## 권장 컷 비율
3-4분 영상 기준:
- 이어가기 컷: 55-70%
- 새 장면 컷: 30-45%

이 비율은 고정값이 아니다. 곡과 플롯에 따라 바꾼다.

## FlowWorker 원칙
- 작업은 오로지 FlowWorker로 실행한다.
- Codex/그루브PD가 Flow 사이트에서 반복 프롬프트 입력을 하지 않는다.
- 새 프로젝트는 곡 단위로 만든다.
- 프로젝트명 예:
  `Afterglow_001_After_Work_Apartment_Groove`
- 프롬프트와 결과물은 곡별 폴더에 저장한다.

## 컷 구성
3-4분 곡 기준:
- 8초 컷 20-30개
- 모든 컷을 같은 인물/장소로만 만들지 않는다.
- 도입, 전개, 반복 훅, 브리지 느낌, 엔딩 여운으로 나눈다.

컷 예:
```text
S001: 밤의 아파트 복도, 문틈으로 새는 초록빛
S002: 낡은 자동차 붉은 벨벳 시트, 라디오 불빛
S003: 흑백 패션 필름처럼 서 있는 남녀 셋
S004: 비 내린 창문에 번진 네온
S005: 손으로 넘기는 아날로그 스토리보드
```

## 금지
- 가사 1:1 시각화 금지
- 같은 템플릿 반복 금지
- 가사 그대로 번역한 장면 금지
- 자동 반복 포맷 금지
- 무의미한 이미지 슬라이드쇼 금지
- 라이브를 스팸처럼 대량 개설 금지
- 자동댓글/봇으로 참여를 인위적으로 부풀리는 운영 금지

## 라이브 운영
- 라이브는 수익화 전 시청시간 확보에 도움이 될 수 있지만, 채널 품질을 해치면 위험하다.
- 라이브 하나당 명확한 콘셉트가 있어야 한다.
- 같은 루프 영상과 같은 음악을 무작정 복제하지 않는다.
- 수익화 전에는 12시간 이내 단위로 끊는 것이 안전하다.
- 자동 채팅 봇은 과도하게 쓰지 않는다. 인위적 참여 조작으로 보이면 위험하다.
