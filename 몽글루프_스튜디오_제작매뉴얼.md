# 몽글루프 스튜디오 제작 매뉴얼

## 목적
이 문서는 몽글루프 스튜디오의 실제 제작 순서를 고정하기 위한 매뉴얼이다.
핵심은 Flow/Veo에서 짧은 영상을 만들고, 확장하고, 마지막 프레임을 다음 시작 프레임으로 넘기면서 3-4분 롱폼을 이어 만드는 것이다.

## 전체 구조
몽글루프 영상은 한 번에 3-4분을 만드는 방식이 아니다.
16초짜리 조각을 여러 개 만들고, 마지막 프레임으로 다음 조각을 이어서 긴 영상처럼 만든다.

기본 루프:
1. Flow 홈 `https://labs.google/fx/ko/tools/flow`로 들어간다.
2. `+ 새 프로젝트`를 누른다.
3. 프로젝트 이름을 입력한다.
   예: `몽글루프_001_딸기케이크_ASMR`
4. 몽글이 기본 캐릭터 이미지와 음식/컨셉 레퍼런스를 첨부한다.
5. 컨셉에 맞는 첫 이미지 `S001.png`를 만든다.
6. `S001.png`를 시작 프레임으로 넣고 8초 영상을 만든다.
7. 이 첫 영상은 `V001`로 관리한다.
8. `V001`을 `Veo 3.1 - Fast`로 확장한다.
9. 확장본을 다운로드한다.
10. 확장본 마지막 프레임을 캡처해서 `S002.png`로 저장한다.
11. `S002.png`를 다음 시작 프레임으로 넣고 다음 프롬프트를 작성한다.
12. 다음 8초 영상 `V002`를 만든다.
13. 다시 확장하고 다운로드한다.
14. 마지막 프레임을 `S003.png`로 저장한다.
15. 3-4분이 될 때까지 반복한다.

번호 체인:
```text
S001 시작 이미지 -> V001 8초 영상 -> V001 확장본 16초 -> S002 마지막 프레임
S002 시작 이미지 -> V002 8초 영상 -> V002 확장본 16초 -> S003 마지막 프레임
S003 시작 이미지 -> V003 8초 영상 -> V003 확장본 16초 -> S004 마지막 프레임
```

## 영상 길이 계산
- 16초 조각 10개 = 160초 = 2분 40초
- 16초 조각 12개 = 192초 = 3분 12초
- 16초 조각 15개 = 240초 = 4분

추천:
- 첫 완성 목표: 12개 조각, 약 3분 12초
- 안정적인 롱폼 목표: 15개 조각, 약 4분

## 파일 이름 규칙
몽글루프는 S번호로 장면을 관리한다.

예시:
- `S001.png`: 첫 시작 이미지
- `V001.mp4`: `S001.png`에서 만든 첫 16초 완성 영상
- `S002.png`: `V001.mp4`의 마지막 프레임
- `V002.mp4`: `S002.png`에서 만든 두 번째 16초 완성 영상
- `S003.png`: `V002.mp4`의 마지막 프레임

원칙:
- 이미지는 `S###.png`
- 완성 영상은 `V###.mp4`
- `V###.mp4`에서 나온 다음 시작 프레임은 `S다음번호.png`로 저장한다.
- `@S###` 레퍼런스 파일과 체인용 `S###` 파일은 헷갈리지 않게 관리한다.

## 새 프로젝트 규칙
- 몽글루프 정식 제작은 기존 Flow 프로젝트 안에서 시작하지 않는다.
- 매 영상 단위로 새 프로젝트를 만든다.
- 프로젝트명은 제작 로그와 같은 이름을 쓴다.
- 예:
  - `몽글루프_001_딸기케이크_ASMR`
  - `몽글루프_002_초코도넛_ASMR`
  - `몽글루프_003_붕어빵_ASMR`
- 새 프로젝트에서 `S001` 이미지 생성부터 시작해야 이전 프로젝트 에셋과 섞이지 않는다.

## 준비물
### Flow
- Google Flow 프로젝트
- Veo 사용 가능 계정
- 가능하면 Ultra 요금제
- `Veo 3.1 - Fast` 모델 사용

### FlowWorker
- 저장 폴더 지정
- `다음 시작 프레임 추출` 버튼 사용
- 프롬프트 파일에 S번호별 프롬프트 작성

## 운영 안전 원칙
- 루프PD/Codex는 콘텐츠 생산 목적으로 Flow 사이트를 직접 반복 조작하지 않는다.
- Flow 사이트에서 생성/확장/다운로드를 자동화할 때는 반드시 FlowWorker를 사용한다.
- 이유: Codex가 직접 브라우저에서 프롬프트 입력/생성을 하루 수십 번 반복하면 자동화 매크로처럼 보일 위험이 있다.
- 사용자가 직접 Flow 화면을 확인하거나 수동으로 조작하는 것은 가능하다.
- Codex는 FlowWorker 코드, 설정, 프롬프트 파일, 저장 폴더, 로그, 결과물 검수만 담당한다.
- FlowWorker는 이 위험을 줄이기 위해 사람 같은 입력/클릭, 워커별 프로필, 로그 기반 검증 구조로 만든다.
- 예외: FlowWorker 기능 수정/강화/업그레이드 검증을 위한 제한적 Flow 화면 관찰과 소량 테스트는 가능하다.

### Suno
- 영상 분위기에 맞는 배경음악 생성
- 음악은 영상보다 먼저 방향을 잡아도 되고, 완성 후 붙여도 된다.

## 1단계: 첫 시작 이미지 만들기
첫 시작 이미지는 전체 영상의 얼굴이다.

FlowWorker 작업 기준:
1. 새 프로젝트를 만든다.
2. 몽글이 캐릭터 기본 이미지와 음식 레퍼런스를 첨부한다.
3. 이미지 모드에서 `S001` 프롬프트를 실행한다.
4. 결과물을 다운로드해서 `S001.png`로 저장한다.
5. `S001.png`가 마음에 들 때만 비디오 체인을 시작한다.

첫 이미지에 반드시 들어가야 하는 것:
- 주인공 캐릭터
- 먹을 음식 또는 여행 장소
- 전체 분위기
- 카메라 거리
- 조명
- 배경

먹방 예시:
- 하얀 몽글 캐릭터
- 딸기 케이크
- 따뜻한 주방
- 나무 테이블
- 부드러운 조명
- 카메라는 정면 중간 거리

여행 예시:
- 하얀 몽글 캐릭터
- 작은 가방
- 일본 골목
- 벚꽃
- 아침 햇살
- 카메라는 캐릭터를 따라가는 낮은 시선

## 2단계: 첫 8초 영상 만들기
Flow에서 시작 이미지를 넣고, 아주 작은 행동만 프롬프트로 쓴다.

중요:
- 한 프롬프트에는 한 행동만 넣는다.
- 너무 많은 행동을 넣으면 장면이 튄다.
- 첫 8초는 세계관 소개라서 안정감이 가장 중요하다.

좋은 프롬프트:
```text
하얀색의 작고 포근한 몽글 캐릭터가 따뜻한 주방 테이블 앞에 앉아 딸기 케이크를 바라본다. 작은 포크를 천천히 들어 올리며 첫 한입을 먹을 준비를 한다. 카메라는 고정되어 있고 아주 천천히 가까워진다. 부드러운 조명, 귀엽고 평온한 분위기.
```

나쁜 프롬프트:
```text
케이크를 먹고 춤추고 웃고 점프하고 카메라가 돌면서 장면이 바뀐다.
```

## 3단계: 확장하기
Flow에서 8초 영상이 마음에 들면 바로 다음을 한다.

순서:
1. 완성된 8초 영상 화면으로 들어간다.
2. `확장` 버튼을 누르기 전에 모델 선택을 확인한다.
3. 기본값이 `Veo 3.1 - Lite`일 수 있으므로 `Veo 3.1 - Fast`로 바꾼다.
4. `확장`을 누른다.
5. 8초가 더 생성되어 약 16초 영상이 된다.

주의:
- 확장 전 모델이 중요하다.
- Lite로 확장하면 퀄리티나 움직임이 약할 수 있다.
- Fast는 현재 제작 속도와 품질 밸런스가 좋다.

### FlowWorker 확장 기능
FlowWorker에는 확장 기능을 붙인다.

1. `현재 영상 확장`
   - 사용자가 Flow에서 확장할 영상 상세 화면을 열어둔다.
   - FlowWorker에서 `현재 영상 확장`을 누른다.
   - FlowWorker가 `Veo 3.1 - Fast` 모델을 맞추고 `확장`을 누른다.
   - 설정된 확장 대기시간 후 현재 상세 화면의 영상을 다운로드한다.
   - 이 기능은 확장 DOM을 안정화하기 위한 소량 검증에도 쓴다.

2. `생성 후 확장`
   - 비디오 모드 설정에 있는 옵션이다.
   - 켜면 FlowWorker가 8초 생성 후 결과 카드를 열고 확장한다.
   - 확장 대기 후 확장 영상을 다운로드한다.
   - 기본값은 꺼져 있다.
   - 실제 대량 제작 전에 반드시 1개 영상으로 소량 테스트한다.

3. `마지막 프레임 자동저장`
   - 비디오 다운로드가 끝나면 FlowWorker가 자동으로 마지막 프레임을 추출한다.
   - `V001.mp4` 또는 `S001.mp4`에서 나온 다음 시작 프레임은 `S002.png`로 저장한다.
   - `V002.mp4`에서 나온 다음 시작 프레임은 `S003.png`로 저장한다.
   - 루프PD는 이 프레임을 보고 다음 프롬프트를 설계한다.
   - 이 기능을 쓰려면 `ffmpeg` 또는 Python 패키지 `imageio-ffmpeg`가 필요하다.

권장 사용 순서:
```text
처음에는 사용자가 영상 상세 화면을 열고 `현재 영상 확장`으로 테스트
-> 확장 버튼/모델/다운로드가 안정적이면 `생성 후 확장`을 켜서 자동 루프 사용
```

## 4단계: 16초 영상 다운로드
확장된 16초 영상이 완성되면 다운로드한다.

저장명:
- 첫 번째 영상이면 `S001.mp4`
- 두 번째 영상이면 `S002.mp4`
- 번호가 꼬이면 나중에 체인 추적이 어려워진다.

저장 폴더:
- 프로젝트별 폴더를 만든다.
- 예:
  `C:/Users/jaekw/Pictures/몽글루프/딸기케이크_001/`

## 5단계: 마지막 프레임 추출
FlowWorker에서 `다음 시작 프레임 추출` 버튼을 사용한다.

사용 방법:
1. FlowWorker를 실행한다.
2. 저장 폴더가 현재 프로젝트 폴더로 되어 있는지 확인한다.
3. `다음 시작 프레임 추출` 버튼을 누른다.
4. 방금 다운로드한 `S001.mp4`를 선택한다.
5. 저장 이름을 확인한다.
6. 기본 추천이 `S002.png`이면 그대로 저장한다.

예시:
- 입력: `S001.mp4`
- 출력: `S002.png`

이 `S002.png`가 다음 영상의 시작 프레임이다.

## 6단계: 마지막 프레임 확인
추출된 마지막 프레임을 반드시 눈으로 확인한다.

확인할 것:
- 캐릭터 얼굴이 망가지지 않았는가
- 음식 상태가 자연스러운가
- 손/입/도구가 이상하게 꼬이지 않았는가
- 다음 장면으로 이어갈 수 있는 자세인가
- 카메라 구도가 너무 가까워지거나 멀어지지 않았는가

나쁜 마지막 프레임이면:
- 그 16초 조각을 다시 만든다.
- 억지로 이어가지 않는다.
- 마지막 프레임이 무너지면 다음 조각도 무너진다.

## 7단계: 다음 시작 프레임으로 넣기
Flow에 `S002.png`를 업로드하거나 에셋으로 선택해서 다음 시작 프레임에 넣는다.

원칙:
- `S002.png`는 이전 영상의 마지막 장면이다.
- 다음 프롬프트는 이 장면에서 바로 이어지는 행동이어야 한다.
- 갑자기 장소, 표정, 음식 상태를 크게 바꾸면 안 된다.

## 8단계: 다음 프롬프트 쓰기
다음 프롬프트는 “이전 마지막 프레임에서 이어지는 아주 작은 행동”이어야 한다.

예시:
```text
이전 장면에서 이어진다. 하얀 몽글 캐릭터가 딸기 케이크 가까이에 앉아 작은 포크로 케이크 옆면을 아주 조금 떠서 천천히 한입 더 먹는다. 케이크는 조금 줄어들고, 캐릭터는 눈을 살짝 감으며 만족스럽게 미소 짓는다. 카메라는 안정적으로 고정되어 있고 부드럽게 아주 조금 가까워진다.
```

다음 장면에서 넣으면 좋은 표현:
- 이전 장면에서 이어진다.
- 같은 캐릭터
- 같은 장소
- 같은 조명
- 음식이 조금 줄어든다.
- 한입 더 먹는다.
- 천천히
- 작고 자연스러운 움직임
- 카메라 고정

피해야 할 표현:
- 갑자기
- 빠르게
- 역동적으로
- 다른 장소로 이동
- 큰 표정 변화
- 카메라가 회전한다
- 장면 전환

## 9단계: 반복
위 과정을 계속 반복한다.

반복 구조:
```text
S001.png -> S001.mp4 -> S002.png
S002.png -> S002.mp4 -> S003.png
S003.png -> S003.mp4 -> S004.png
...
S012.png -> S012.mp4 -> S013.png
```

12개 정도 만들면 3분대 롱폼이 된다.

## 10단계: 최종 합치기
16초 영상들이 모이면 영상 편집툴에서 이어붙인다.

추천:
- 처음에는 단순 이어붙이기
- 너무 많은 전환 효과 금지
- 흐름이 어색한 곳만 아주 약한 디졸브 사용
- 음악은 전체 감정선을 하나로 묶는 역할

파일 예시:
```text
S001.mp4
S002.mp4
S003.mp4
...
S012.mp4
```

최종 파일:
```text
Mongloop_StrawberryCake_EP01_3min.mp4
```

## 먹방형 프롬프트 진행 예시
### S001
캐릭터가 케이크를 바라보고 첫 한입을 준비한다.

### S002
첫 딸기를 천천히 먹는다.

### S003
맛을 음미하고 작게 웃는다.

### S004
케이크 옆면을 조금 떠서 먹는다.

### S005
크림이 묻은 포크를 보고 다시 한입 먹는다.

### S006
케이크가 조금 줄어든 상태에서 딸기를 하나 더 먹는다.

### S007
접시를 살짝 가까이 당긴다.

### S008
케이크 가운데 부분을 한입 먹는다.

### S009
눈을 감고 천천히 씹는다.

### S010
케이크가 절반 정도 줄어든다.

### S011
마지막 딸기 조각을 바라본다.

### S012
마지막 한입을 먹고 행복하게 웃는다.

## 여행형 제작 순서
여행형은 먹방보다 먼저 기획이 필요하다.

먼저 정할 것:
1. 나라
2. 도시
3. 계절
4. 시간대
5. 캐릭터 복장
6. 이동 동선
7. 마지막 감정

예시: 교토 벚꽃 산책
```text
S001 여관 앞에서 출발
S002 조용한 골목을 걷는다
S003 벚꽃 아래에서 멈춘다
S004 작은 디저트 가게를 발견한다
S005 딸기 모찌를 산다
S006 강가 벤치에 앉는다
S007 모찌를 한입 먹는다
S008 다리를 건넌다
S009 노을 골목을 걷는다
S010 밤 조명이 켜진 거리를 바라본다
S011 숙소로 돌아온다
S012 창가에서 하루를 마무리한다
```

## 품질 체크리스트
각 16초 조각마다 확인한다.

- 캐릭터가 같은 캐릭터로 보이는가
- 얼굴이 귀여운가
- 손/입/도구가 망가지지 않았는가
- 음식/장소 상태가 이어지는가
- 카메라가 너무 흔들리지 않는가
- 다음 시작 프레임으로 쓸 수 있는 마지막 장면인가
- 16초 동안 한 가지 행동만 했는가
- 반복해서 봐도 편안한가

## 실패 기준
아래 중 하나라도 있으면 다시 만든다.

- 캐릭터가 다른 캐릭터처럼 변함
- 얼굴이 망가짐
- 손가락/포크/숟가락이 심하게 이상함
- 음식이 갑자기 사라짐
- 장면이 갑자기 바뀜
- 카메라가 크게 흔들림
- 마지막 프레임이 다음 장면으로 이어지기 어려움

## 좋은 장면 기준
- 소리를 꺼도 이해된다.
- 한입 먹는 행동이 자연스럽다.
- 캐릭터가 귀엽다.
- 음식이 맛있어 보인다.
- 배경이 따뜻하다.
- 다음 장면이 궁금하다.
- 반복 재생해도 피곤하지 않다.

## 최종 업로드 전 체크
- 제목이 명확한가
- 썸네일에서 캐릭터와 음식/장소가 바로 보이는가
- 영상 초반 5초가 귀여운가
- 음악이 너무 시끄럽지 않은가
- 영상이 3분 이상인가
- 좋은 구간을 숏폼으로 자를 수 있는가

## 한 줄 운영 철학
몽글루프는 장면을 많이 만드는 프로젝트가 아니라, 귀여운 순간을 끊기지 않게 이어서 사람들이 계속 보고 싶게 만드는 프로젝트다.
