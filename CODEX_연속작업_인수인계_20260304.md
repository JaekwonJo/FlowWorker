# FlowWorker 인수인계

## 최신 기준
- 날짜: 2026-04-29
- 현재 목표: `Flow Worker = Flow Classic Plus의 이미지 워커 + S자동화 워커만 참고해서, 그 두 개를 간단하게 통합한 새 앱`
- 현재 사용자 용어:
  - 이미지 워커 = 이미지 모드
  - 예전 S자동화 워커 = 비디오 모드
  - 단, 비디오 모드의 작동 방식은 기존 S자동화 워커 그대로 구현

## 지금 상태
- 문서 기준은 `이미지 워커 / S자동화 워커만 참고`로 다시 고정했다.
- `prompt_parser.py`는 `S001>S002 Prompt -> S001 Prompt`, `V001>V002 Prompt -> V001 Prompt` 규칙을 반영했고, 비디오용 시작/끝 프레임은 `frame_start_tag`, `frame_end_tag`로 따로 보존한다.
- `legacy_worker_bridge.py`는 숨은 원본 이미지 워커/예전 S자동화 워커 엔진을 IPC로 띄우는 구조로 연결되어 있다.
- 이미지 모드는 실제 사용자 테스트에서 `레퍼런스 첨부 -> 생성 -> 1K 다운로드 저장` 성공 로그까지 확인했다.
- `FlowWorker` 색상은 이제 파란 계열이 아니라 브라운/앰버 계열로 바꿨다.
- 대기열은 여러 카드가 한 줄에 보이는 구조로 다시 바꿨고, 백엔드 queue 상태 서명이 같으면 다시 그리지 않게 바꿔 깜빡임을 줄였다.
- 비디오 프롬프트는 사용자 파일 기준 `V001 Prompt`, `V001>V002 Prompt` 형식을 그대로 읽고, 내부 브리지가 런타임 파일에서만 `S001>S002 Prompt : V001 Prompt : ...` 형태로 바꿔 원본 비디오 워커에 넘긴다.

## 절대 헷갈리면 안 되는 점
- 참고 원본은 `Flow Classic Plus 전체`가 아니다.
- 오직 아래 두 개만 본다.
  - `이미지 워커`
  - `S자동화 워커`

## 다음 작업 우선순위
1. `비디오 모드` 실사용 테스트를 돌려서 `V001 Prompt`, `V001>V002 Prompt` 경로가 실제 원본 비디오 워커에 맞게 동작하는지 확인한다.
2. 비디오 모드에서 시작/끝 프레임 첨부와 다운로드까지 실제 로그를 기준으로 보정한다.
3. 이미지/비디오 둘 다 대기열 카드, 상태 요약, 로그 문구를 최종 사용자 체감 기준으로 더 다듬는다.
4. 각 단계마다 문서와 시작 프롬프트를 갱신한다.
