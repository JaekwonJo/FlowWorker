# FlowWorker 인수인계

## 최신 기준
- 날짜: 2026-04-29
- 현재 목표: `Flow Worker = Flow Classic Plus의 이미지 워커 + S자동화 워커만 참고해서, 그 두 개를 간단하게 통합한 새 앱`

## 지금 상태
- `FlowWorker` 저장소에서 넓게 베껴왔던 기능 이식 커밋들은 대부분 되돌렸다.
- 현재 코드는 다시 가벼운 독립 뼈대 상태에 가깝다.
- 문서 기준도 이제 `이미지 워커 / S자동화 워커만 참고`로 다시 고정했다.
- `prompt_parser.py`는 새 기준으로 다시 손봐서, `S001>S002 Prompt`도 실제 입력은 `S001 Prompt`로, `V001>V002 Prompt`도 실제 입력은 `V001 Prompt`로 정리하도록 바꿨다.
- 비디오 라우트는 `frame_start_tag`, `frame_end_tag` 메타데이터를 따로 보존하도록 바꿨다.

## 절대 헷갈리면 안 되는 점
- 참고 원본은 `Flow Classic Plus 전체`가 아니다.
- 오직 아래 두 개만 본다.
  - `이미지 워커`
  - `S자동화 워커`

## 다음 작업 우선순위
1. `이미지 워커`의 실제 생성 → 다운로드 흐름을 다시 읽는다.
2. `S자동화 워커`의 실제 비디오 흐름을 다시 읽는다.
3. 그 두 흐름만 합쳐서 `FlowWorker` 새 엔진에 단계별로 옮긴다.
4. 각 단계마다 문서와 시작 프롬프트를 갱신한다.

## 현재 코드 방향
- 유지: 독립 실행 구조, 독립 설정 파일, 독립 브라우저 프로필, 독립 UI 뼈대
- 폐기: 넓은 `Flow Classic Plus` 기준으로 급히 끌어왔던 중간 이식 흔적

## 다음 Codex가 먼저 할 일
- `FlowWorker` 코드를 바로 늘리기 전에, `Flow Classic Plus` 안에서 실제 `이미지 워커`와 `S자동화 워커` 경로를 다시 정확히 집는다.
- 그 뒤 `FlowWorker` 쪽에 필요한 최소 기능부터 다시 넣는다.
