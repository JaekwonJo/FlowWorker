# FlowWorker 인수인계 문서

## 최신 작업 시간

- 2026-04-27 Asia/Seoul

## 프로젝트 방향

- `FlowWorker`는 `Flow Classic Plus`를 고치는 프로젝트가 아니다.
- 새 폴더 / 새 저장소 / 새 런타임의 독립 앱이다.
- 단, 기존 제품의 유용한 기능은 로직 단위로 선별 이식한다.

## 현재 완료된 것

- 새 독립 저장소 생성 및 GitHub 연결 완료
- 새 Tk 기반 Flow Worker 창 생성
- 프로젝트 목록 / 프롬프트 파일 / 저장 폴더 관리
- Edge 원격 디버그 포트 기반 브라우저 연결
- 프롬프트 번호 계획 생성
- VBS 실행 파일로 무콘솔 실행

## 현재 작업 중

- 그록워커와 확실히 다른 색상 테마 적용
- FlowWorker 전용 아이콘 적용
- 이미지 모드 핵심 자동화 1차 이식
  - 브라우저 페이지 확보
  - 입력창 찾기
  - 이미지 모드 강제 맞춤
  - 생성 개수 / 화질 맞춤
  - 사람처럼 프롬프트 타이핑
  - 제출 버튼 클릭
- 그록워커와 충돌하지 않게 Edge 기본 연결도 분리
  - 기본 프로필: `runtime/flow_worker_edge_profile`
  - 기본 포트: `127.0.0.1:9333`
- FlowWorker 이미지 프롬프트 입력은 이제 `fill`이 아니라 한 글자씩 타이핑 방식이다.
- 새 FlowWorker 설정 로더는 이제 기존 Flow Classic Plus의 이미지 워커1 / S자동화 워커1 설정에서 핵심 값들을 직접 가져온다.
  - `start_url`
  - `input_selector`
  - `submit_selector`
  - `download_image_quality`
  - `download_video_quality`
  - `generate_wait_seconds`
  - `next_prompt_wait_seconds`
  - `edge_window_*`
- FlowWorker 상단 `현재 상태` 칸은 이제 남은 시간을 실시간으로 보여준다.
  - 생성 후 다운로드 대기
  - 다음 작업 대기
- FlowWorker 이미지 모드는 이제 생성 후 이미지 다운로드까지 한 번에 시도한다.
  - 카드 위치 추정
  - 더보기 버튼 찾기
  - 다운로드 메뉴 클릭
  - 품질 선택
  - Playwright download 이벤트를 받아 저장 폴더로 저장
- FlowWorker 프롬프트 입력은 이제 Flow Classic Plus 방식의 inline 레퍼런스 첨부를 일부 이식했다.
  - `@S001` 같은 태그를 만나면 본문을 그냥 끝까지 타이핑하지 않고,
  - `@`로 검색창 호출
  - 레퍼런스 검색창 탐지
  - `오래된 순` 정렬
  - 태그 입력
  - `Enter` 첨부
  - 그 뒤 본문 타이핑 재개
- FlowWorker 브라우저 탭 정리도 보강했다.
  - 대상 프로젝트 탭이 이미 있으면 그 탭을 우선 유지
  - 남는 `about:blank` 탭은 닫도록 보정
- FlowWorker는 이제 실행마다 `logs/action_trace_*.log` 파일을 만든다.
  - 레퍼런스 `@` 호출
  - 검색창 후보 덤프
  - 정렬 클릭
  - Enter 첨부
  같은 세부 행동을 따로 남긴다.
- FlowWorker 대기열 카드는 이제 한 줄에 여러 개 보이도록 그리드 배치로 바꿨다.
