# 몽글루프 스튜디오 병렬 제작 설계

## 목적
몽글루프 스튜디오는 사용자가 “오늘 영상 몇 개 만들어주세요”라고 요청하면, Codex 총괄 프로듀서가 기획/설계/프롬프트/작업 분배를 정리하고 Flow Worker 여러 개가 동시에 제작하는 구조를 목표로 한다.

## 운영 안전 원칙
- Codex/루프PD가 콘텐츠 생산 목적으로 Flow 사이트를 직접 반복 조작하지 않는다.
- 모든 Flow 생성 작업은 FlowWorker를 통해 실행한다.
- Codex는 워커별 설정, 프롬프트 파일, 저장 폴더, 로그 분석, 결과물 검수만 담당한다.
- Flow 화면에서 직접 수동 판단이 필요하면 사용자가 한다.
- 병렬 제작도 Codex가 브라우저 여러 개를 직접 만지는 것이 아니라, Flow Worker1/2/3에 작업을 배정하는 방식으로 한다.
- 예외: FlowWorker 기능 수정/강화/업그레이드 때는 Codex가 Flow 화면을 제한적으로 확인하고 소량 테스트할 수 있다.

## 역할 구조
### 사용자
- 최종 창작자
- 방향 제시
- 취향 승인
- 완성본 최종 판단

### Codex 총괄 프로듀서
- 아이디어 기획
- 시리즈 선정
- 캐릭터 설정
- 장소 동선 설계
- 감정선 설계
- 샷 리스트 작성
- Flow 프롬프트 작성
- 작업 우선순위 결정
- 병렬 워커 배정
- 결과물 검수 기준 제시

### Flow Worker들
- Flow Worker1: 1번 영상 또는 1번 시리즈 담당
- Flow Worker2: 2번 영상 또는 2번 시리즈 담당
- Flow Worker3: 3번 영상 또는 후보 장면 테스트 담당
- Flow Worker4+: 대량 생산 또는 다른 프로젝트 병렬 담당

## 사용자가 말하는 이상적인 명령
예:
```text
총괄 프로듀서님, 오늘 몽글루프 영상 3개 만들어주세요.
```

그러면 Codex는 바로 실행하지 않고 먼저 아래를 정리한다.

1. 오늘 만들 영상 후보 제안
2. 각 영상의 목적
3. 캐릭터/장소/음식/감정선
4. 16초 단위 샷 리스트
5. FlowWorker 배정
6. 예상 소요 시간
7. 실패 시 대체안

## 병렬 제작 기본 원칙
- 각 워커는 서로 다른 Edge 프로필과 CDP 포트를 사용한다.
- 각 워커는 서로 다른 저장 폴더를 사용한다.
- 각 워커는 서로 다른 프롬프트 파일을 사용한다.
- 같은 Flow 프로젝트를 동시에 건드릴 때는 에셋/다운로드 검색 충돌을 피해야 한다.
- 가능하면 영상별 프로젝트 폴더를 분리한다.

## 워커별 책임
### Flow Worker1
- 가장 중요한 메인 롱폼 담당
- 실패하면 즉시 수정
- 가장 좋은 프롬프트와 가장 안정적인 설정 사용

### Flow Worker2
- 두 번째 롱폼 또는 같은 포맷의 변형 담당
- Worker1과 다른 음식/장소를 맡는다.

### Flow Worker3
- 테스트 담당
- 여행형, 새로운 캐릭터, 새 음식, 새 카메라 구도 검증
- 실패해도 메인 제작에 피해가 없게 한다.

## 병렬 제작에서 가장 중요한 문제
### 1. 로그인/세션 문제
각 워커의 Edge 프로필이 Flow에 로그인되어 있어야 한다.

### 2. 다운로드 충돌
모든 워커가 같은 폴더에 저장하면 파일명이 꼬일 수 있다.
그래서 워커별/영상별 저장 폴더를 분리한다.

### 3. 에셋 검색 충돌
Flow 안에서 `S001`, `S002` 같은 이름이 여러 프로젝트에 섞이면 헷갈린다.
영상별 프로젝트 또는 명확한 에셋 관리를 해야 한다.

### 4. 프롬프트 번호 충돌
Worker1의 `S001`과 Worker2의 `S001`은 서로 다른 폴더에 있어야 한다.
Flow 안에서는 프로젝트를 분리하거나, 최소한 검색/다운로드 시 정확히 구분해야 한다.

### 5. Ultra 큐 관리
동시에 너무 많이 넣으면 실패 원인을 추적하기 어렵다.
처음에는 2-3개 워커부터 안정화하고, 이후 4개 이상으로 늘린다.

## Codex가 해야 할 제작 진행 방식
### 1. 기획 회의
사용자와 오늘 만들 영상 방향을 정한다.

### 2. 제작안 작성
각 영상별로 아래를 만든다.
- 제목 후보
- 썸네일 방향
- 캐릭터 설정
- 장소/음식 설정
- 16초 단위 샷 리스트
- 프롬프트 초안
- FlowWorker 배정

### 3. 작업 시작 전 확인
- 저장 폴더
- 프롬프트 파일
- Flow 프로젝트 URL
- 워커 프로필
- 생성 개수
- 비디오 품질
- 생성 후 대기시간

### 4. 실행
각 워커를 실행한다.

### 5. 검수
각 16초 조각마다 마지막 프레임이 다음 장면에 쓸 수 있는지 확인한다.

### 6. 재작업
망가진 조각은 다시 만든다.

### 7. 최종 편집
CapCut 등에서 이어붙이고 음악을 넣는다.

## 1차 구현 목표
현재 FlowWorker는 여러 워커를 띄우는 구조가 일부 있다.
몽글루프용 병렬 제작을 위해 다음을 확실히 해야 한다.

1. 워커별 프로필/포트 자동 분리
2. 워커별 저장 폴더 자동 분리
3. 워커별 프롬프트 파일 자동 분리
4. 워커별 진행 상태 한눈에 보기
5. 마지막 프레임 추출을 워커별 폴더 기준으로 실행
6. 다운로드 후 자동으로 다음 `S###.png` 생성
7. 실패한 워커만 재시작
8. 오늘 제작 대시보드

## 확장 기능과 병렬 제작
- 각 워커는 비디오 생성 후 `생성 후 확장` 옵션을 켤 수 있어야 한다.
- 확장 대기시간은 워커별로 다르게 둘 수 있다.
- 초기에는 Worker1에서만 확장 기능을 소량 검증한다.
- 안정화 후 Worker2, Worker3에 확장 옵션을 켠다.
- 확장 영상 다운로드 후 마지막 프레임 추출까지 자동으로 이어지는 것이 최종 목표다.
- `마지막 프레임 자동저장` 옵션으로 다운로드 직후 `S다음번호.png`를 만든다.
- 다음 구현 목표는 이 `S다음번호.png`를 Flow에 자동 업로드/첨부하는 기능이다.

## 오늘 제작 대시보드 개념

```text

Worker2 | 일본푸딩 EP01   | S002 확장 중 | 1/12 완료

완료 영상: 0
제작 중: 3
실패 조각: S003 Worker1
다음 할 일: S003 재생성
```

## 운영 철학
병렬 제작은 단순히 많이 돌리는 것이 아니다.
Codex 총괄 프로듀서가 오늘의 제작 목표를 정하고, 각 Flow Worker에게 역할을 나누고, 결과를 검수해서 하나의 채널 전략으로 묶는 것이다.
