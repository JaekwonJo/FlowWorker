# 몽글루프 스튜디오 제작 회의록

## 2026-05-07 1차 제작 방향 회의

### 프로젝트명
- `몽글루프 스튜디오`

### 이름 체계
- 브랜드/스튜디오: `몽글루프 스튜디오`
- 대표 캐릭터 1차 이름: `몽글이`
- Codex 총괄 프로듀서 별칭: `루프PD`

### 몽글루프 뜻
- `몽글`: 포근하고 귀엽고 마음이 부드러워지는 감성
- `루프`: 마지막 프레임을 다음 시작 프레임으로 이어가는 제작 방식, 그리고 계속 보게 되는 반복 시청성

### 프로젝트 방향
- 음식 하나에 묶이지 않는다.
- 먹방, 여행, 음악, 명소, 계절 이벤트, 여러 캐릭터, 여러 채널까지 확장 가능한 AI 영상 제작 스튜디오로 간다.
- 우선은 하나의 성공 포맷을 터뜨린다.
- 1개월 안에 100만 조회수급 영상 1개를 목표로 본다.
- 한국어 운영 + 일본 성공 포맷 겨냥으로 시작한다.

### 역할 고정
- 사용자: 원작자, 취향 결정자, 최종 승인자
- 루프PD: 총괄 프로듀서/감독
- 루프PD는 프롬프트만 쓰지 않고, 기획/카메라/장소 동선/감정선/반복 행동/샷 리스트/프롬프트/자동화까지 설계한다.

### 제작 방식
- 8초 영상 생성
- `Veo 3.1 - Fast`로 확장
- 16초 영상 다운로드
- 마지막 프레임 추출
- 다음 `S###.png` 시작 프레임으로 저장
- 그 프레임을 보고 다음 행동 프롬프트 작성
- 반복해서 3-4분 롱폼 제작

### 프롬프트 운영 결정
- 즉흥적으로 마지막 프레임을 보고 다음 프롬프트를 짜도 된다.
- 단, 짠 프롬프트는 반드시 파일에 저장한다.
- 이유:
  - 어떤 프롬프트가 성공했는지 남겨야 한다.
  - 어떤 프롬프트가 실패했는지 남겨야 한다.
  - 다음 영상에서 재사용 가능한 표현을 축적해야 한다.
  - 실패 패턴을 줄이고 성공 패턴을 빠르게 늘려야 한다.

### 병렬 제작 방향
- Flow Worker1, Flow Worker2, Flow Worker3를 제작팀처럼 운영한다.
- 워커별 프로필/포트/저장폴더/프롬프트 파일이 충돌하지 않게 관리한다.
- 사용자는 “오늘 영상 몇 개 만들어주세요”라고 요청할 수 있어야 한다.
- 루프PD는 먼저 제작 회의를 열고, 영상 후보/샷 리스트/워커 배정/예상 시간/리스크를 정리한다.

### 유튜브 채널
- 유튜브 채널 개설이 필요하다.
- 채널명 후보는 `몽글루프`, `Mongloop Studio`, `몽글루프 스튜디오`를 기준으로 검토한다.
- 영상 안 언어는 최소화하고, 제목/설명/태그에서 한국어 + 영어 + 일본 감성 키워드를 조합한다.

### 다음 할 일
1. 워커별 충돌 없는 실행 구조 정리
2. 프롬프트/결과 기록 양식 만들기
3. 첫 번째 영상 제작 로그 생성
4. 유튜브 채널 개설 준비 체크리스트 작성
5. 첫 대표 캐릭터 `몽글이` 설정 확정

## 2026-05-07 2차 채널 포지션 회의

### 확정 방향
- 처음 영상 10개 이상은 먹방 ASMR 중심으로 간다.
- 여행/명소/음악 시리즈는 확장 가능성으로 열어두지만, 초반 채널 정체성은 먹방 ASMR이다.
- 채널 감성은 “틀어놓고 작업해도 좋고, 소리만 들어도 좋고, 가끔 보면 귀여워서 힐링되는 채널”이다.
- 우선은 귀여움으로 승부한다.

### 음악 방향
- 잔잔한 로파이 음악을 기본으로 한다.
- ASMR 느낌은 가끔 섞는다.
- Flow가 뽑는 원본 소리는 사용자가 편집 단계에서 수동으로 조절한다.
- Suno AI는 결제되어 있으므로 루프PD가 음악 프롬프트를 제공하고 사용자가 후보곡을 생성한다.

### 몽글이 캐릭터 결정
- 대표 포인트는 꽃핀이다.
- 현재 계절이 봄이므로 목도리보다 꽃핀이 어울린다.
- 분홍 볼, 말 없음, 표정만으로 감정 표현을 유지한다.
- 단, 매 영상마다 시그니처 의상/소품은 조금씩 바꾼다.
- 이유: 매번 완전히 똑같으면 자동화 채널처럼 보일 수 있다.

### 첫 영상
- 001은 딸기 케이크 ASMR로 확정.
- 소재는 대전 명물로 유명한 딸기시루 스타일의 큰 딸기 케이크.
- 공개 업로드에서는 실제 브랜드 로고/상표를 화면에 직접 넣지 않는다.
- 제목/설명에서는 필요하면 “대전 명물 딸기시루 스타일”처럼 표현한다.

### 다음 영상
- 002는 아직 미정.
- 갑자기 일본으로 이동하지 않는다.
- 우선 유행하는 음식/유명한 음식 중심으로 고민한다.

### 운영 결정
- 프롬프트는 마지막 프레임을 보고 즉흥적으로 조정할 수 있다.
- 하지만 모든 프롬프트는 반드시 제작 로그에 저장한다.
- 성공/실패 원인을 기록해서 몽글루프만의 성공 표현 라이브러리를 만든다.

### Flow 운영 안전 결정
- 루프PD/Codex는 콘텐츠 생산 목적으로 Flow 사이트를 직접 반복 운영하지 않는다.
- 대량 Flow 실행은 반드시 FlowWorker를 통해서만 한다.
- 이유: Codex가 직접 Flow 사이트에서 프롬프트 입력/생성을 하루 수십 번 반복하면 자동화 매크로처럼 보일 위험이 있다.
- FlowWorker를 잘 만들어둔 이유가 바로 이 운영 안전성과 재현성을 확보하기 위해서다.
- 루프PD는 기획/프롬프트/설정/로그/검수/자동화 코드 개선에 집중한다.
- 예외: FlowWorker 기능 수정/강화/업그레이드 검증을 위한 제한적 접속과 소량 테스트는 가능하다.

### FlowWorker 직원 교육 기록
- FlowWorker는 몽글루프 제작팀의 실행 직원으로 본다.
- 루프PD는 FlowWorker가 어떤 업무를 할 수 있게 되었는지 별도 교육 로그에 기록한다.
- 교육 로그 파일:
  `몽글루프_스튜디오_FlowWorker직원교육로그.md`
- 기록 대상:
  - 새로 가르친 기능
  - 테스트 결과
  - 아직 실패하거나 미검증인 부분
  - 다음 교육 과제

## 2026-05-07 3차 FlowWorker 확장 기능 회의

### 결정
- 몽글루프 롱폼 루프에는 Flow의 `확장` 기능이 필수다.
- 현재 FlowWorker에는 확장 전용 기능이 없으므로 추가해야 한다.

### 목표 루프
```text
FlowWorker로 8초 영상 생성
-> FlowWorker로 확장 실행
-> 16초 영상 다운로드
-> 루프PD가 마지막 프레임 추출
-> 그 이미지를 다음 시작 프레임으로 사용
-> 다음 프롬프트를 저장하고 다시 FlowWorker 실행
```

### 구현 방향
- `현재 영상 확장` 버튼을 추가한다.
  - Flow 상세 화면의 현재 영상을 확장한다.
  - 확장 기능 DOM을 소량 검증하는 용도다.
- 비디오 설정에 `생성 후 확장` 옵션을 추가한다.
  - 켜면 8초 생성 후 자동으로 결과 카드를 열고 확장한다.
  - 기본값은 꺼둔다.
- `확장 대기(초)` 설정을 추가한다.
  - 기본값은 75초다.
- `마지막 프레임 자동저장` 옵션을 추가한다.
  - 영상 다운로드 직후 마지막 프레임을 `S다음번호.png`로 저장한다.
  - 루프PD는 직접 캡처하지 않고 이 자동 저장본을 보고 다음 프롬프트를 짠다.

### 운영 주의
- 대량 생성은 직접 브라우저 조작이 아니라 FlowWorker로만 한다.
- 기능 검증은 소량으로만 한다.
- 다음 병목은 로컬 `S###.png`를 Flow에 업로드하고 시작 프레임으로 붙이는 기능이다.
