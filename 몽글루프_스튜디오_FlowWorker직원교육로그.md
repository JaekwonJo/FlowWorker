# 몽글루프 스튜디오 FlowWorker 직원 교육 로그

이 문서는 FlowWorker를 몽글루프 제작팀의 실행 직원으로 교육한 기록이다.
루프PD가 어떤 기능을 가르쳤고, 어디까지 검증했으며, 다음에 무엇을 훈련해야 하는지 추적한다.

## 교육 원칙
- FlowWorker는 콘텐츠 생산 실행 담당이다.
- 루프PD는 기획, 프롬프트, 샷 설계, 자동화 개선, 결과 검수 담당이다.
- 대량 생성은 Flow 사이트 직접 반복 조작이 아니라 FlowWorker를 통해 실행한다.
- 기능 개선 검증은 소량 테스트만 허용한다.
- 모든 테스트 결과는 로그와 제작 기록에 남긴다.

## Grok Worker 참고 메모
- `Grok Worker`의 안정 운영 원칙도 결국 “사람이 먼저 전용 Edge를 열고 로그인한 뒤, 워커가 `connect_over_cdp`로 붙기”였다.
- 과거 Grok Worker에서도 자동 Edge 실행/종료는 로그인 차단 문제가 있어 운영 원칙에서 제외된 기록이 있다.
- FlowWorker도 콘텐츠 생산 운영에서는 전용 Edge 프로필/포트를 먼저 정상 로그인 상태로 만든 뒤 붙는 방식을 우선한다.
- FlowWorker 코드 개발 중 소량 테스트에서는 자동 실행을 쓸 수 있지만, 서비스 차단 페이지로 튕기면 로그인/계정 상태 문제로 보고 작업을 멈춘다.
- 파일 첨부 방식은 Grok Worker와 유사하게 우선 `input[type=file].set_input_files`를 시도하고, 실패하면 `+`/업로드 버튼과 file chooser로 폴백한다.

## 2026-05-07 교육 004: 새 프로젝트에서만 제작 시작

### 결정
- 몽글루프 제작 테스트는 기존 프로젝트 상세 URL에서 시작하지 않는다.
- 기본 진입 URL은 `https://labs.google/fx/ko/tools/flow`로 둔다.
- FlowWorker는 필요 시 Flow 홈에서 새 프로젝트 버튼을 눌러 새 프로젝트 안으로 들어간 뒤 작업한다.
- 새 프로젝트명은 제작 단위 이름으로 입력한다.
  - 예: `몽글루프_001_딸기케이크_ASMR`
  - 테스트 예: `몽글루프_001_딸기케이크_ASMR_TEST`

### 이유
- 기존 프로젝트 안에서 테스트하면 이전 에셋, 이전 결과물, 기존 설정이 섞인다.
- 시작 프레임 첨부 여부를 착각하기 쉽다.
- 새 프로젝트로 시작해야 한 편의 제작 체인과 로그가 깔끔하게 분리된다.

### 추가 안전장치
- 시작 프레임 첨부는 Playwright 파일 입력 성공만으로 성공 처리하지 않는다.
- composer 주변 썸네일/첨부 칩 증가를 확인한 뒤에만 프롬프트를 입력한다.
- 첨부 검증 실패 시 프롬프트 입력과 제출을 중단한다.

## 2026-05-07 교육 005: 인간적인 타이핑

### 문제
- 기존 FlowWorker 타이핑은 오타 시뮬레이션이 있어도 전체 리듬이 일정해서 기계적으로 보일 수 있었다.
- 특히 긴 한국어 프롬프트를 거의 같은 박자로 입력하면 사람처럼 보이지 않는다.

### 교육 내용
- 입력 전 짧게 멈춰 프롬프트를 읽는 느낌을 준다.
- 10-28자 단위로 불규칙하게 멈칫한다.
- 공백, 줄바꿈, 문장부호 뒤에는 더 길게 쉰다.
- 한국어 조사와 어미 뒤에도 아주 짧은 대기를 넣는다.
- 한국어 오타는 같은 글자를 한 번 더 치는 방식이 아니라, 다른 한국어 글자를 입력한 뒤 백스페이스로 지운다.
- `mixed` 모드가 아니면 붙여넣기는 쓰지 않고 직접 타이핑한다.

### 권장 설정
- 정식 제작 테스트는 타이핑 속도 `x3` 또는 `x4` 권장
- 대량 작업에서 속도가 필요하면 `x5`까지 사용
- `x8` 이상은 검증 작업 외에는 권장하지 않는다.

### 상태
- 코드 반영 완료
- 실제 Flow 화면에서 체감 리듬 확인 필요
