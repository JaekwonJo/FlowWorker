# FlowWorker Codex 설명서

이 폴더에서 코덱스가 지켜야 할 가장 중요한 기준은 아주 단순합니다.

## 1. 이 프로젝트의 목적
- `Flow Worker`는 새 독립 프로그램이다.
- 참고 원본은 오직 아래 두 개뿐이다.
  - `Flow Classic Plus`의 `이미지 워커`
  - `Flow Classic Plus`의 `S자동화 워커`
- 그 외 `Flow Classic Plus` 전체 구조나 다른 기능은 기준으로 삼지 않는다.

## 2. 작업 기준
- `이미지 워커`에서 실제로 하던 생성 + 다운로드 흐름을 그대로 본다.
- `S자동화 워커`에서 실제로 하던 번호 처리 + 비디오 흐름을 그대로 본다.
- 두 워커를 가볍게 합쳐서 `Flow Worker` 하나로 만드는 것이 목표다.
- "비슷하게"가 아니라, 가능하면 사용자가 체감하던 순서와 동작을 그대로 맞춘다.

## 3. 금지 기준
- `Flow Classic Plus 전체`를 넓게 참고하지 않는다.
- 메인 통합 앱의 다른 기능을 끌어오지 않는다.
- 이어달리기, 파이프라인, 안 쓰는 보조 기능을 기준으로 삼지 않는다.
- 원본 기준이 애매하면 반드시 `이미지 워커 / S자동화 워커` 안에서 다시 찾는다.

## 4. 작업 방식
- 수정 전에는 무엇을 바꿀지 짧게 먼저 말한다.
- 설명은 쉬운 한국어로 한다.
- 코드 수정 후에는 아래 문서를 같이 갱신한다.
  - `README.md`
  - `CODEX_연속작업_인수인계_20260304.md`
  - `새 Codex 시작용 한방 프롬프트.txt`
- 가능하면 코드 수정 + 문서 수정 + `git commit` + `git push`를 한 번에 마무리한다.

- 2026-04-29 기준으로, 예전에 `Flow Classic Plus`를 넓게 참고하며 가져왔던 `FlowWorker` 코드와 문서는 되돌리기 시작했다.
- 지금부터는 `이미지 워커`와 `S자동화 워커`만 떼서 다시 합친다.
- 새 기능을 만들 때도 먼저 원본 두 워커 안에서 실제 동작 경로를 확인한 뒤 옮긴다.
